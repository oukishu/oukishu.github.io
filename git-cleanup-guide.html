Git Repo History Reset & Slimming Guide

Designed for bloated codebases, accidental large file commits, and purging obsolete history

⚠️ High-Risk Operation Context

CRITICAL WARNING: This workflow will permanently purge all commit history on this branch, both locally and on GitHub. Ensure your current working directory contains the exact code state you want to preserve before proceeding, and notify your teammates beforehand.

This method breaks the timeline by creating an "Orphan Branch," turning your current codebase state into a single, clean Initial commit that overwrites the master branch.

Phase 1: Core Flow - Reset the Main Branch (e.g., main)

  1. Create and switch to a new, history-free orphan branch This branch carries all current files over but starts with zero commit history pointers. 
    git checkout --orphan latest_branch
  2. Stage all current files 
    git add -A
  3. Commit the changes as your new starting point 
    git commit -m "Initial commit"
  4. Delete the old, bloated main branch locally 
    git branch -D main
  5. Rename the current orphan branch back to main 
    git branch -m main
  6. Force push to GitHub to overwrite the remote history 
    git push -f origin main

Phase 2: Cleanup Flow - Completely Drop Obsolete master Branches

If you have legacy master branches left over from an old repository structure, purge them cleanly using the following steps:

  1. Delete the master branch locally 
    git branch -D master
  2. Delete the master branch from the remote GitHub repository 
    git push origin --delete master
    💡 Troubleshooting (Permission / Branch Protection Error):
    If step 2 fails, it means master is still designated as the Default branch or is governed by a Branch protection rule in your GitHub repository.
    Solution: Go to the GitHub web interface for the repo, navigate to Settings -> Branches, switch the Default branch to main, remove protection rules on master, and re-run the delete command.

Phase 3: Final Step - Zero-Pain Local Directory Deep Slimming

Even though the pointers are cleanly reset on GitHub, your local .git/objects/ directory still retains dangling object fragments and historical bloat cached in your system. To shrink your local storage to absolute minimum size without wrestling with manual garbage collection:

The Ultimate Clean Slate Maneuver:
1. Once Phase 1 and Phase 2 are successfully pushed (confirming GitHub is clean and fresh).
2. Completely delete your local project root folder. (This wipes the bloated .git cache instantly).
3. Clone the clean repo back down to your machine: 
git clone <your-repository-url>
Your newly cloned project folder is now pure, optimized down to the byte, and entirely clear of dead-weight legacy commits!
